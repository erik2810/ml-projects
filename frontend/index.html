Guide & Instructions

Complete walkthrough of every feature in the Graph ML Research toolkit. Each section explains what to do, what to expect, and what to observe in the results.

1 — GNN Explorer

Node classification on Zachary’s Karate Club using GCN or GAT layers, built from scratch.

How to use

  1. Load the graph. Click Load Karate Club. A force-directed graph appears with 34 nodes and 78 edges. Nodes are coloured by ground-truth community (Mr. Hi vs Officer).
  2. Configure training. Choose GCN or GAT as the layer type. Adjust hidden dimension (default 16), layers (default 2), learning rate (0.01), and epochs (200).
  3. Train the model. Click Train Model. Watch the loss curve descend in real time. When training finishes, test accuracy and F1 score appear in the metrics panel.
  4. Predict on nodes. Enter comma-separated node IDs (e.g. 0,16,33) and click Predict. A table shows each node’s predicted class and confidence percentage.

What to observe

  • Loss curve: Should drop steeply in the first 50 epochs, then plateau. GAT typically converges slightly faster than GCN due to attention-weighted aggregation.
  • Accuracy: Expect 90–97% test accuracy. The classic split uses nodes 0–7 as the test set.
  • Graph colouring: After training, the graph colours update to show predicted classes. Misclassified nodes (typically nodes 2, 8, or 9 near the community boundary) stand out.
  • GAT vs GCN: GAT learns to weight important neighbours differently — try both and compare F1 scores. GAT often captures the bridge nodes (e.g. node 2) more accurately.

Theory

A GCN layer computes H(l+1) = σ(ÃH(l)W(l)) where Ã is the symmetrically normalised adjacency with self-loops. GAT replaces fixed normalisation with learnable attention coefficients αij = softmax(LeakyReLU(aT[Whi || Whj])).

2 — Graph Generator

Conditional graph VAE that generates graphs matching target structural properties.

How to use

  1. Train the generator. Set hidden dim (64), learning rate (0.001), and epochs (500). Click Train Generator. The model learns to reconstruct and generate small graphs conditioned on structural features.
  2. Set conditions. Use the three sliders to specify target properties:
    • Nodes (5–20): number of nodes in the generated graph
    • Density (0.05–0.95): fraction of possible edges present
    • Clustering (0–1.0): target transitivity coefficient
  3. Generate. Choose how many samples (1–24) and click Generate Graphs. A gallery grid shows each generated graph with its measured statistics.

What to observe

  • Condition adherence: Generated graphs should roughly match the requested node count, density, and clustering. Exact match is unlikely — the model learns a continuous distribution.
  • Diversity: With the same conditions, multiple samples should look structurally different while sharing similar statistics. This shows the VAE is sampling from a learned latent space, not memorising.
  • High-density graphs: Set density above 0.7 to see nearly-complete graphs. Set it below 0.15 to see sparse, tree-like structures.
  • Clustering control: High clustering + moderate density produces triangle-rich “community-like” structures. Low clustering + high density produces random-looking dense graphs.
  • Graph cards: Each card displays n (nodes), e (edges), d (measured density) for quick comparison with your target.

3 — VAE / Diffusion

Graph variational autoencoder and discrete denoising diffusion for generative modelling.

Graph VAE

  1. Train. Configure latent dim (8), hidden dim (32), and click Train VAE. The VAE encodes graphs into a continuous latent space and decodes via a sigmoid adjacency reconstruction.
  2. Generate. Click Generate from VAE to sample random graphs from the learned prior N(0,I). A gallery of generated graphs appears.
  3. Interpolate. Set source and target graph indices, choose interpolation steps (3–12), and click Interpolate. A strip shows smooth geometric morphing between two training graphs.

Diffusion Model

  1. Train. Set timesteps (100), hidden dim (64), and click Train Diffusion. The model learns to denoise progressively corrupted graph structures.
  2. Generate. Click Generate from Diffusion. The reverse diffusion process iteratively denoises random noise into coherent graph structures.

What to observe

  • VAE loss: Comprises reconstruction loss + KL divergence. Watch for the KL term initially collapsing near zero (“posterior collapse”) before rising as the model learns meaningful latent representations.
  • Interpolation strip: The transition from graph A to graph B should be smooth — edges gradually appear/disappear and node positions blend linearly. This demonstrates the geometric interpolation with spectral layout matching.
  • VAE vs Diffusion quality: Diffusion-generated graphs often look more realistic (fewer isolated nodes, better connectivity) but take longer to train. VAE is faster but may produce “blurry” graphs with ambiguous edge probabilities.
  • Latent space: The architecture diagram on the right shows the encoder–latent–decoder pipeline. Encoder uses 2-layer GCN, decoder uses an MLP on concatenated node pairs to predict edge probability.

4 — Spatial 3D

Autoregressive tree VAE for generating 3D spatial trees and neuron morphologies, plus geometric mesh interpolation.

Tree Generation

  1. Choose data type. Select “Branching tree” for abstract fractal trees or “Neuron morphology” for biologically-inspired dendritic structures.
  2. Generate synthetic data. Click Generate Synthetic to create training examples. The gallery shows 3D isometric projections of trees with colour-coded node types.
  3. Train the model. Click Train Spatial. The VAE or diffusion model learns the joint distribution over tree topology and 3D positions.
  4. Generate new trees. Click Generate to sample novel trees from the learned distribution.
  5. Analyse. Click Analyze to compare generated tree statistics against the training distribution.

Mesh Interpolation

  1. View meshes. Click View Meshes to load 6 canonical shapes: Cube, Octahedron, Icosahedron, Hexagonal Prism, Star, and Torus.
  2. Train mesh VAE. Click Train Mesh VAE to learn a latent space over mesh geometry.
  3. Generate. Sample new mesh shapes from the VAE prior.
  4. Interpolate. Select two shapes (e.g. Cube→Icosahedron) and click Interpolate. A strip shows the mesh smoothly morphing between the two shapes.

What to observe

  • Isometric projection: Trees are rendered with x, y, z axis indicators in the corner. Rotate mentally using these references — the projection uses cos(30°)/sin(30°) mapping.
  • Node types: Large nodes = soma (root), medium = branch points (2+ children), small = tips (terminal). Colour intensity indicates depth from root.
  • Morphological stats: Each tree card shows n (nodes), e (edges), br (branch points), tips (terminal tips), S (Strahler order — a measure of branching complexity borrowed from hydrology).
  • Mesh interpolation strip: Labelled t=0 to t=1. Watch how node positions blend linearly while edge topology transitions — edges from the source shape fade out while edges from the target shape fade in.
  • Neuron morphologies: Should show characteristic dendritic branching patterns with 3–6 primary branches splitting into finer arborisations. The Strahler order should be 2–4 for realistic neurons.

5 — Physics GNN

Physics-informed graph neural networks with discrete differential geometry: cotangent Laplacians, reaction-diffusion dynamics, curvature attention, and energy regularisation.

Getting started

  1. Choose a scenario. Select one of four physical systems:
    • Heat Diffusion on Lattice: 7×7 grid graph. Nodes are labelled Cold/Warm/Hot based on distance from a heat source. Tests whether the GNN learns smooth temperature fields.
    • Curvature Classification: Random 3D point cloud. Nodes are labelled Flat/Curved/Sharp based on local discrete curvatures. Tests geometric feature extraction.
    • Elastic Stress Analysis: Lattice graph under simulated elastic deformation. Nodes labelled Low/Medium/High stress. Tests physics-aware message passing.
    • Molecular Structure: Random 3D molecular graph with bond lengths. Nodes labelled by local curvature. Tests the model on non-grid geometry.
  2. Generate the system. Click Generate System. The graph appears in the visualisation panel with colour-coded labels and a legend.
  3. Configure physics components. Toggle the five checkboxes to enable/disable:
    • Curvature features: Appends 6 discrete curvatures (mean, Gaussian, κ1, κ2, shape index, curvedness) as node features.
    • Multi-scale diffusion: CotangentConv layers using geometric edge weights from vertex positions.
    • Reaction-diffusion: ReactionDiffusionLayer implementing Gray-Scott dynamics within the GNN.
    • Curvature attention: CurvatureAttention that biases message passing based on local curvature similarity.
    • Energy regularisation: Adds Dirichlet energy + total variation + elastic energy as learnable regularisation terms to the loss.
  4. Train. Click Train Physics GNN. Watch the loss curve descend. When done, the prediction graph appears alongside accuracy metrics.

Analysis tools

  1. Ablation Study. Click Run Ablation Study. Trains 7 model variants, systematically disabling each physics component. Results appear in a table showing accuracy and parameter count for each configuration. Expect the “Full model” row to score highest.
  2. Compute Energy. Click Compute Energy. Shows three energy functionals evaluated on the current graph signal:
    • Dirichlet energy: Measures signal smoothness. Low = smooth, high = oscillating.
    • Total variation: Edge-preserving measure. Highlights sharp transitions.
    • Elastic energy: Shape rigidity. High = large deformations from rest.
  3. Analyse Curvature. Click Analyse Curvature. Displays min/max/mean for all 6 discrete curvature types. Flat regions have near-zero curvature; corners of a lattice have high values.
  4. Heat Diffusion. Set a source node index and click Visualise Diffusion. A 6-frame strip appears showing heat spreading from the source over time (blue→orange→red). The diffusion uses the cotangent Laplacian heat kernel exp(-tL).
  5. Reaction-Diffusion. Set feed rate (default 0.055) and kill rate (0.062), then click Run Gray-Scott. A 6-frame strip shows Turing pattern emergence: initially uniform concentrations evolve into spots, stripes, or labyrinthine structures depending on the feed/kill ratio.

What to observe

  • Accuracy: With all components enabled, expect 90–97% on heat diffusion, 85–92% on curvature, 80–90% on stress. Molecular is hardest (75–88%).
  • Ablation impact: Removing curvature features typically causes the largest accuracy drop (~5–10%). Removing energy regularisation causes the smallest drop but increases overfitting on longer training runs.
  • Heat diffusion strip: Heat should spread concentrically from the source node. On a grid lattice, the pattern forms a diamond shape (L1 norm wavefront). On molecular graphs, it follows bond topology.
  • Gray-Scott patterns: With default parameters (f=0.055, k=0.062), you get spot-like Turing patterns. Try f=0.04, k=0.06 for stripes, or f=0.025, k=0.06 for labyrinthine patterns.
  • Graph colouring: The initial graph uses a 3-class colour scheme: blue (Cold/Flat/Low), orange (Warm/Curved/Medium), red (Hot/Sharp/High). After training, the prediction graph uses the same scheme for comparison.
  • Molecular graph: Renders as a 3D isometric molecular structure. Bond lengths create non-uniform edge weights in the cotangent Laplacian, which the model leverages for classification.

Theory

The cotangent Laplacian Lij = −wij (off-diagonal), Lii = Σj wij uses geometric edge weights from vertex positions. The Gray-Scott reaction-diffusion system dA/dt = Da∇²A − AB² + f(1−A) produces Turing instabilities when the activator diffuses slower than the inhibitor (Da < Db).

6 — Hyperbolic GNN

Graph neural networks in hyperbolic space — embedding hierarchical structures in the Poincaré ball with Riemannian optimization.

How to use

  1. Generate a graph. Choose a graph type — Binary Tree works best for demonstrating hyperbolic advantages. Click Generate Graph. The graph appears in the Poincaré disk with the root near the centre and leaves near the boundary.
  2. Run the simulation. Adjust curvature (1.0 default), spring strength, and charge parameters. Click Start Simulation. Watch nodes rearrange themselves under spring-mass forces on the curved manifold. The energy readout should decrease over time.
  3. Train the GNN. Configure hidden dim, layers, and click Train Hyperbolic GNN. The model performs message passing in hyperbolic space using logmap/expmap operations. Watch the loss curve and accuracy metrics.
  4. Compare embeddings. Click Compare Euclidean vs Hyperbolic. Two side-by-side visualizations show the same graph embedded in flat space vs curved space. The distortion table shows how much the embeddings distort the original graph distances.

What to observe

  • Tree structure: Binary trees embed perfectly in hyperbolic space (zero distortion) because hyperbolic space grows exponentially — matching the exponential growth of tree branches. Euclidean embeddings must distort such structures.
  • Boundary effect: Nodes near the Poincaré disk boundary are actually very far apart in the hyperbolic metric, even though they appear close visually. The dashed boundary circle represents infinity.
  • Geodesic arcs: Edges appear as circular arcs, not straight lines. These arcs are always orthogonal to the boundary circle — the shortest paths in hyperbolic geometry.
  • Simulation energy: Should decrease monotonically. Nodes spread out to minimize spring energy while respecting the hyperbolic metric. The final layout reveals the hierarchical structure.
  • Distortion comparison: For trees, expect hyperbolic distortion near 0 and Euclidean distortion > 0.5. For grid graphs, both distortions will be similar since grids don't have hierarchical structure.
  • Curvature parameter: Higher curvature compresses the disk — nodes pack tighter near the boundary. Lower curvature makes the space more Euclidean-like. Try c=0.5 vs c=2.0 to see the difference.

Theory

The Poincaré ball model represents n-dimensional hyperbolic space inside the unit ball. The metric tensor gx = (2/(1−||x||²))² · I makes distances grow exponentially near the boundary. Message passing uses logmap to project neighbours into the tangent space at each node, aggregates in flat tangent space, then maps back via expmap. RiemannianAdam adapts gradient descent to the manifold by rescaling gradients with the inverse metric tensor.

7 — WL Test

Interactive 1-Weisfeiler-Leman colour refinement test for graph isomorphism and GNN expressivity.

How to use

  1. Select a graph pair. Choose from three examples:
    • Star K1,5 vs Path P6: Easily distinguished — WL separates them in iteration 1 because the star has a node with degree 5.
    • C6+short chord vs C6+long chord: Two 6-cycles with chords placed differently. WL distinguishes them after 2–3 iterations because the neighbour multisets eventually diverge.
    • C6 vs C3∪C3: A single 6-cycle versus two disjoint 3-cycles. Easily distinguished — degree distributions are identical (all degree 2) but WL finds the difference in 2 iterations via 2-hop neighbourhood structure.
  2. Set iterations (1–10). More iterations = finer colour refinement.
  3. Run. Click Run WL Test. An interactive stepper appears with two graph panels and colour histograms.
  4. Step through. Use the Previous/Next buttons, slider, or play button to animate through each WL iteration.

What to observe

  • Node colours: Each iteration assigns new colours. Initially, nodes with the same degree share a colour. As iterations progress, colours split based on increasingly fine neighbourhood structure.
  • Histogram bars: Below each graph pair, a histogram compares colour distributions. When the histograms diverge, a green “Graphs distinguished!” badge appears.
  • Convergence: After enough iterations, the colour partition stabilises (no more splits). At this point, additional iterations have no effect — the WL test has reached its maximum discriminative power.
  • GNN connection: The number of WL iterations needed to distinguish two graphs equals the minimum GNN depth required. This is a direct consequence of the Xu et al. (2019) theorem: standard message-passing GNNs are at most as powerful as 1-WL.
  • Colour count: The info panel shows how many distinct colours each graph has. When the graphs have different colour counts, they are provably non-isomorphic.

General Tips

Workflow

  • Sequential dependency: Most tabs require generating data before training, and training before predicting. Buttons that require a prior step are disabled until that step is complete.
  • Training times: GNN and Generator train in seconds. VAE and Spatial models take 10–30 seconds. Physics ablation studies train 7 models sequentially and take 1–2 minutes.
  • In-memory state: All models and data are held in server memory. Refreshing the page resets the state and you need to regenerate/retrain.
  • Loss curves: Every training panel shows a real-time loss curve. A healthy curve drops steeply early and plateaus. If the curve oscillates wildly, reduce the learning rate.

Implementation details

  • All from scratch: Every model (GCN, GAT, VAE, Diffusion, Spatial Tree VAE, Physics GNN, WL) is implemented from scratch in PyTorch — no PyTorch Geometric, DGL, or other graph libraries.
  • Visualisation: All graphs are rendered with D3.js. Force-directed layouts use D3 force simulation; spatial/physics graphs use isometric projection; mesh interpolation uses spectral positioning.
  • Backend: FastAPI serves the REST API. Each tab has its own route module. Models run on CPU for portability.
